1 redis sds实现

#sds (simple dynamic string)

note

总结

1、长度惰性释放，预申请，减少内存重新分配次数。
2、常数获取字符串长度，二进制安全，可以处理字符串中字符=0的情况。
3、sds结构__packed__属性，方便访问header信息。（压缩空间？）
4、兼容部分C字符串, typedef char *sds; 一般除了对\0有需求的C字符串函数都可以用。

一、结构

header: len, alloc, flags
len：字符串实际长度
alloc：实际申请空间大小
flags：结构体类型，有sdshdr64， sdshdr32. p.s: sdshdr5, 不知道干嘛的。。
*根据申请的空间大小来控制使用数值类型, e.g. sdshdr32 (1 << 16 <= initlen < 1ll << 32).

二、内存管理

zmalloc
1) PREFIX_SIZE: 如果系统有malloc_size函数，malloc申请内存的时候会将大小写在前面
2）update_zmalloc_stat_alloc 内存对齐
e.g. new_n = n + (8 - n%8) = n + (n - n&7)
3) update_zmalloc_stat_add 更新静态变量used_memory

三、Api

介绍部分函数
1、sds sdsnewlen(const void *init, size_t initlen)
新建一个sds对象

params：
1）init如果为空，初始化sds字符串地址空间为0；
2）如果init不为空&initlen值不等于0，复制init为起始的initlen字节

adventure：
1）申请内存（hdrlen + initlen +1),1字节用于初始化的字符串末尾补'\0', 安全（可以兼容部分字符串函数）。
2）sh->len保存字符串长度，即时字符串中间包含'\0'字符，也可以获取完整的字符串值

2、sds sdscatvprintf(sds s, const char *fmt, va_list ap);
1）功能: 可变参数格式化连接sds
2）sds sdsMakeRoomFor(sds s, size_t addlen);
功能为原有字符串扩容
修改header类型，惰性扩容
