8 redis ziplist实现

note

  • 单个值为占用空间比较少的数据内容
  • 根据数据类型不同，选择不同的存储结构
  • 使得小内存数据密集排列，减少对内存打散的压力

总结

结构

11开头: 表示后面content是整数, encoding占用一字节
1100 16位
1101 32位
1110 64位
11110000 24位
11111110 8位
(11110000，11111110) 13个数来表示 0-12

下述结构体对上述格式数据解码可得
typedef struct zlentry {
unsigned int prevrawlensize, prevrawlen; //prevrawlensize = 1 or 5 用于描述prevrawlen占空间大小,
unsigned int lensize, len; //lensize = 1 or 2 or 5 用于描述len占空间大小，第一个字节的前2位来标识coding格式，
unsigned int headersize; //prevrawlensize + lensize
unsigned char encoding; //coding
unsigned char *p; //前面一个entry
} zlentry;

更新当前节点之后，会导致后面节点都可能被更新prelensize，导致结构的变化。以下是连锁更新后续的节点。
更新的复杂度是 O(n^2), 更新一个节点会resize后面的节点，然后会引起后面的节点继续更新。
某资料上说：实际使用起来效果还不错。
