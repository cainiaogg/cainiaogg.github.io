9 redis object实现

对于list结构的实现
照着书看源码，看到ziplist。
不过目前看的版本3.2.9使用的是quicklist。

对于有序集合的实现
ziplist和zset（skiplist & dict 查找精确值O（1），有序值O（log（n）））

  • 类型转换

总结

对象类型转换

长度小于20 && 可以转换成long，1）如果值小于10000 用初始好的[1,10000]整型变量 2）如果原值为raw类型（sds），释放并类型至为int，返回
长度小于44，使用embstr存（无法修改）

数据结构

redisObject

type

对象类型，string、list、set、zset、hash

encoding

对象编码，不同的对象数据会选择不同的编码方式。根据数据类型和大小不同，会使用不同的结构。比如较短字符串在新建后无修改的的情况，会使用embstr，后面细说。

常见指令

string

type=OBJ_STRING
encoding=OBJ_ENCODING_INT、OBJ_ENCODING_RAW、OBJ_ENCODING_EMBSTR

string这个东西底层编码可以是 整数、sds、简单动态字符串embstr

embtr这个东西是为了一次malloc申请sds和redisObject两个内存，
正常的操作是sds单独申请一次，redisObject单独申请一次，总共两次

list

type=OBJ_LIST
encoding=OBJ_ENCODING_QUICKLIST、OBJ_ENCODING_ZIPLIST(redis3.2.9貌似没用。。)

set

type=OBJ_SET
encoding=OBJ_ENCODING_HT、OBJ_ENCODING_INTSET
如果集合里的所有数连在一起可以用long long存下，则使用inset，否则使用dict

zset

type=OBJ_ZSET
encoding=OBJ_ENCODING_SKIPLIST 、OBJ_ENCODING_ZIPLIST

//为啥OBJ_ENCODING_SKIPLIST同时用跳表和字典实现呢？ 原因：字典o(1)查找key-value，跳表维护zset的有序性。

###hash
type=OBJ_HASH
encoding=OBJ_ENCODING_HT 、OBJ_ENCODING_ZIPLIST
